1. Методы выборки (Получение данных / Поиск)

Эти методы используются для извлечения записей из базы данных. Почти все они возвращают либо объект модели, либо коллекцию (Collection).

    all($columns = ['*']) — Получает все записи из таблицы. Можно передать массив конкретных колонок.

    get($columns = ['*']) — Выполняет сформированный запрос и возвращает коллекцию результатов.

    find($id) — Ищет запись по её первичному ключу (id). Если не найдено — вернет null.

    findOrFail($id) — Ищет по id. Если записи нет — выбрасывает исключение ModelNotFoundException (автоматически отдает ошибку 404 на фронтенд).

    first() — Возвращает первую запись, соответствующую условиям запроса.

    firstOrFail() — Возвращает первую запись или выбрасывает 404 ошибку, если ничего не найдено.

    firstWhere($column, $operator = null, $value = null) — Быстрый хелпер: делает where() и сразу возвращает first().

    value($column) — Получает значение только одной конкретной колонки из первой найденной записи (минуя создание объекта модели).

    pluck($column, $key = null) — Возвращает коллекцию, состоящую только из значений указанной колонки. Если передан $key, то коллекция будет ассоциированной (ключ => значение).

2. Методы создания и сохранения (Insert / Update)

Методы для добавления новых данных или обновления существующих.

    save(array $options = []) — Универсальный метод. Если объект модели новый — делает INSERT. Если объект был подгружен из базы — делает UPDATE измененных полей.

    create(array $attributes = []) — Создает новую запись в базе и сразу возвращает объект этой модели. Важно: требует настройки $fillable или $guarded в модели (защита от массового заполнения).

    forceCreate(array $attributes) — Создает запись в базе, игнорируя ограничения $fillable / $guarded.

    update(array $attributes = [], array $options = []) — Массово обновляет поля у подгруженной модели или у цепочки запроса.

    push() — Сохраняет не только саму модель, но и все её загруженные связи (отношения).

    touch() — Принудительно обновляет только таймстэмп updated_at текущей записи.

3. Умные методы «Найди или Создай / Обнови»

Помогают избежать написания громоздких конструкций с if/else.

    firstOrCreate(array $attributes, array $values = []) — Ищет запись по $attributes. Если находит — возвращает её. Если нет — создает новую, объединяя параметры из первого и второго массивов.

    firstOrNew(array $attributes, array $values = []) — То же самое, что и firstOrCreate, но если запись не найдена, он не сохраняет её в базу автоматически, а просто возвращает новый объект (нужно вызвать ->save() вручную).

    updateOrCreate(array $attributes, array $values = []) — Ищет запись по $attributes. Если нашел — обновляет её данными из $values. Если не нашел — создает новую.

4. Методы удаления (Delete)

    delete() — Удаляет текущую запись из базы данных (вызывается на объекте модели).

    destroy($ids) — Статический метод. Удаляет записи по их ID без предварительной загрузки моделей. Можно передать один ID, массив destroy([1, 2, 3]) или коллекцию.

    forceDelete() — Принудительно удаляет запись из базы физически, если в модели включено «мягкое удаление» (SoftDeletes).

5. Методы работы с Мягким удалением (Soft Deletes)

Эти методы доступны только если в модели подключен трейт use Illuminate\Database\Eloquent\SoftDeletes;.

    withTrashed() — Включает в выборку как живые, так и «мягко удаленные» записи (у которых заполнено поле deleted_at).

    onlyTrashed() — Выбирает только те записи, которые были ранее мягко удалены.

    restore() — Восстанавливает мягко удаленную запись (очищает поле deleted_at).

    trashed() — Проверяет, удалена ли конкретная модель (возвращает true/false).

6. Проверка состояния модели (Dirty, Clean, Original)

Используются, когда нужно узнать, менялось ли что-то в полях модели во время выполнения скрипта.

    isDirty($attributes = null) — Возвращает true, если какие-то поля модели (или конкретное поле) были изменены в коде, но еще не сохранены в базу.

    isClean($attributes = null) — Противоположность isDirty. Возвращает true, если поля модели остались нетронутыми.

    wasChanged($attributes = null) — Возвращает true, если указанные поля были изменены в текущем цикле запроса во время последнего сохранения (save() или update()).

    getOriginal($attribute = null) — Позволяет получить первоначальные значения полей модели, какими они были при загрузке из базы данных, даже если ты их уже перезаписал в коде.

7. Методы отношений (Relationships / Загрузка связей)

Используются для оптимизации запросов к связанным таблицам (например, User -> posts).

    with($relations) — Жадная загрузка (Eager Loading). Загружает связи одновременно с основной моделью одним-двумя запросами, решая проблему N+1.

        Пример: User::with('posts')->get();

    load($relations) — Ленивая жадная загрузка (Lazy Eager Loading). Подгружает связи для модели, которая уже была выбрана из базы ранее.

        Пример: $user->load('posts');

    loadMissing($relations) — Подгружает связь только в том случае, если она еще не была загружена у этой модели ранее.

    withCount($relations) — Подсчитывает количество связанных записей без загрузки самих записей (добавляет поле {relation}_count).

        Пример: User::withCount('posts')->get(); (у каждого юзера появится поле posts_count).

    has($relation) — Фильтрует модели на основе наличия связи. (Выбрать только тех пользователей, у которых есть хотя бы один пост).

    whereHas($relation, Closure $callback) — Более мощный фильтр. Позволяет наложить условия на связанную таблицу. (Выбрать пользователей, у которых есть посты, опубликованные в этом месяце).

8. Преобразование данных (Сериализация)

    toArray() — Превращает модель (и все её загруженные связи) в обычный массив PHP.

    toJson($options = 0) — Превращает модель в строку JSON.

    replicate() — Клонирует модель. Создает точную копию существующего объекта в памяти, но без id и таймстемпов, подготавливая её к сохранению как новой записи.
